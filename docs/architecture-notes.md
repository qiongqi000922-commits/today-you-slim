# 架构与维护说明

## 近期诉求主线

- 个人记录：固定口令或 QQ 登录，拍照、端侧人脸点位校正、体重和一句话心情记录，记录可删除。
- 趋势展示：体重高低图、涨跌日历、缩略图分页加载、回放弹窗和弹幕式心情/评论展示。
- 社区能力：匿名共享、社区档案、点赞、评论、分享、人气榜单和共享体重高低变化。
- 隐私与管理：首次隐私协议选择、基础/完整模式、撤销签署、注销账户、管理台查看账户/记录/签署审计并删除账户。
- 部署策略：正式站和 `/test` 隔离，当前默认直接部署正式站，明确要求时先发测试环境。

## 当前结构

- `server.mjs`：无框架 Node 服务，负责静态资源、API、QQ OAuth、管理台、JSON 文件存储和照片文件存储。
- `public/shared.js`：浏览器共享基础层，集中放置路径拼接、API 超时请求、WebAuthn 编解码和图片交互判断，主站、管理台和回放页都应优先复用这里的能力。
- `public/app.js`：主站交互、拍照流程、趋势图、社区和设置页。
- `public/replay.js`：回放弹窗/页面，负责图片预载、时间轴和弹幕。
- `public/admin.js`：管理台登录、账户列表、记录和隐私签署审计。
- `public/styles.css` / `public/replay.css` / `public/admin.css`：对应界面的样式。
- `scripts/verify.mjs`：上线前基础检查入口，校验后端和各前端入口脚本语法，并确认关键静态资源版本号已更新。

## 维护边界

- 前端优先使用小型渲染函数做组件抽象，当前已将社区详情拆成 profile、评论项、评论空态和评论区组件。
- 多处共用行为应集中处理。浏览器侧优先进入 `public/shared.js`，例如图片长按/拖拽禁止、API 超时、WebAuthn 编解码；业务页面只保留场景逻辑。
- 后端继续保持无框架部署简单性，但关键边界集中在辅助函数中：安全响应头、请求体限制、安全解码、写入队列、隐私模式校验。
- JSON 文件存储适合当前小规模使用；当账户和照片互动继续增长时，应迁移到 SQLite/PostgreSQL，并把照片迁到对象存储/CDN。

## 下一阶段拆分方向

- 后端：优先把 `server.mjs` 中的纯函数按 `auth`、`records`、`community`、`food`、`moderation`、`admin` 拆入 `src/server/`，每次只移动一类函数并保持 API 行为不变。
- 前端：优先把 `public/app.js` 中的食物记录、社区详情、隐私设置、Passkey 流程拆成独立文件；仍可先用普通脚本和 `window` 命名空间，等模块稳定后再切换 ES module。
- 样式：把 `public/styles.css` 按页面区域添加分区注释，后续拆成 `base.css`、`community.css`、`food.css`、`settings.css` 时以现有选择器为边界，避免跨区覆盖。
- 数据：保持 JSON 写入队列作为短期方案；当月活和照片量上升后，先迁移账户、记录、互动到 SQLite，再迁照片到对象存储。
- 发布：正式部署前运行 `npm run check`，同时确认首页、管理台和回放页的资源版本号同步更新。

## 安全与可靠性策略

- 管理台独立 Cookie、CSRF 令牌、失败锁定和认证事件审计。
- 主站 Cookie `HttpOnly`、`SameSite=Lax`，管理台 `SameSite=Strict`。
- 请求入口限制 Host、URL、Cookie、Content-Length 和同源写请求；只信任本机代理传入的 `X-Forwarded-For`。
- 按 IP、接口类型和登录账户做多级限流，上传、登录、Passkey、文本发布和客户端日志都有单独频率上限。
- 所有 JSON 写入按文件排队，降低多设备同时写入导致覆盖的风险。
- JSON 请求体限制为 256KB，照片上传保持独立上限和 JPG 魔数校验。
- 静态 HTML 带 CSP、nosniff、同源 frame 限制、权限策略和 HSTS。
- 评论、心情、登录口令和资源 ID 都有服务端校验，前端做即时提示但不作为最终信任边界。
- systemd 服务使用只读系统目录、私有临时目录、能力收敛、资源上限和仅数据目录可写的运行约束。

## 体验策略

- 动效保持短、轻、可关闭：系统设置减少动态效果时关闭主要过渡和点赞动画。
- 移动端优先：底部吸底页签、详情页底部弹出、回放 3:4 图片比例、禁止图片长按拖拽。
- 多设备体验：页面回到前台、重新联网或获得焦点时节流同步当前视图，减少手机和电脑之间状态滞后。
