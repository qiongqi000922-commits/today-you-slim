# 可靠性治理技术方案

版本：2026-06-29

本文档描述“今天你瘦了吗”当前阶段已经落地的可靠性治理方案。目标是在不引入额外云数据库成本的前提下，提升数据可恢复性、运行可观测性、合规可解释性和后续维护效率。

## 1. 本地数据库化与备份恢复

当前线上运行仍以 `data/*.json` 作为业务读写源，新增本地 SQLite 镜像作为查询、审计、故障排查和灾难恢复的辅助数据层。

- 默认数据库路径：`data/wellecho.sqlite3`
- 默认备份目录：`data/backups`
- 运行开关：`LOCAL_DB_ENABLED=1`
- 镜像策略：业务 JSON 写入后使用短延迟合并同步，避免每次请求都触发数据库写入。
- 备份策略：服务启动后周期备份，默认 6 小时一次，可在管理台手动触发。
- 降级策略：SQLite 不可用或镜像失败时不阻塞主业务，系统记录运行告警并在管理台展示。
- 恢复策略：优先使用活跃 JSON 数据；如 JSON 损坏，可通过 SQLite 镜像或备份辅助重建关键账户、记录、审核和治理信息。

本地数据库保存结构化快照，包括账户、记录、健康同步摘要、审核、治理、运行日志和性能指标等。会话类临时数据不纳入数据库镜像，降低敏感面和恢复复杂度。

## 2. 合规与数据生命周期

隐私政策和用户协议版本更新为 `2026-06-29`，补充了以下说明：

- 本地 SQLite 镜像和周期备份的用途、范围和保存逻辑。
- 性能体验指标的采集范围，包括启动、模型加载、图片下载和接口耗时。
- 删除、注销、撤销签署后，活跃数据镜像随下一次同步更新，备份按安全和灾难恢复策略轮转。
- 管理台可查看协议签署、账号治理、举报反馈和关键运行事件。

数据生命周期原则：

- 用户主动内容以账户为边界隔离。
- 用户删除记录后，业务 JSON 和对象文件优先删除，数据库镜像随后同步。
- 注销账户后清理个人资料、记录、Passkey、社区足迹和签署状态。
- 安全审计、争议处理或法律法规要求保留的信息只保留必要范围。

## 3. 性能与运行观测

系统已经具备前端性能打点和服务端运行事件记录能力，管理台新增可靠性治理模块，用于集中查看：

- 数据库镜像状态、文件大小、表数量、最后同步时间、最后备份时间。
- 最近备份文件列表。
- 运行健康检查项。
- 下一阶段治理计划。
- 现有性能指标，包括启动、页面就绪、模型加载、图片下载、接口失败和资源加载异常。

建议后续持续补齐的核心体验指标：

- 应用冷启动到可交互时间。
- 首次进入拍照页到模型可用时间。
- 回放图片下载张数、总耗时、失败率和缓存命中率。
- 食物识别接口耗时、失败率和用户取消率。
- AI 总结生成耗时、截断率和缓存命中率。

## 4. 架构与异步解耦

本次改造将数据库镜像能力独立到 `lib/local-db.mjs`，主服务只负责在必要节点触发镜像或备份。

设计原则：

- 业务写入仍走原有 JSON 流程，降低一次迁移风险。
- 数据库镜像异步执行，不阻塞用户请求。
- 镜像和备份失败只产生告警，不影响拍照、回放、登录等核心路径。
- 后续可以在该模块上继续演进为完整 SQLite 主存储，迁移路径清晰。
- 与预测模型、性能分析、运行告警等能力保持数据接口解耦。

服务器承受能力考虑：

- 单机 SQLite 写入量远低于图片上传和 AI 调用成本，适合作为当前阶段的本地数据治理层。
- 镜像写入使用防抖合并，避免高频记录时放大磁盘 IO。
- 备份周期可配置，默认 6 小时，避免频繁复制数据库。
- 管理台手动操作提供即时镜像和备份，便于上线前和问题排查时使用。

## 5. 安全治理与账号风控

当前系统已经具备以下技术基础：

- QQ、Apple、Passkey 登录和绑定。
- CSRF、Host 白名单、HttpOnly Cookie、输入校验和请求限流。
- 文本安全、图片安全、举报反馈和账号冻结、封禁、恢复。
- 登录 IP、User-Agent、认证事件和治理事件记录。
- 管理台集中查看账户、记录、反馈、审核、治理和运行状态。

本地数据库镜像补齐了安全追踪的一层结构化证据能力。发生恶意操作时，可以从 JSON、SQLite 镜像、运行日志和管理台事件中交叉定位来源、影响范围和处置记录。

后续安全治理方向：

- 将高风险操作统一接入二次确认和审计事件。
- 将敏感接口的异常峰值接入管理台图表。
- 对图片、AI、登录、Passkey、健康数据同步分别维护错误率和告警阈值。
- 逐步把管理台操作日志和运行告警从“列表查看”升级为“趋势 + 事件关联”。

## 运维操作

强制刷新数据库镜像：

```bash
curl -X POST https://furby.top/admin/api/reliability/database/mirror
```

手动创建数据库备份：

```bash
curl -X POST https://furby.top/admin/api/reliability/database/backup
```

直接检查 SQLite：

```bash
sqlite3 data/wellecho.sqlite3 ".tables"
sqlite3 data/wellecho.sqlite3 "select key, updated_at from json_documents order by updated_at desc limit 10;"
```

> 管理台接口需要管理员会话，上述命令仅表示路径和动作，实际执行时应带上管理台认证 Cookie。
